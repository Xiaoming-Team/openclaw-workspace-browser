# OpenClaw Workspace 文件浏览器 - 需求文档

## 概述
一个为 OpenClaw 设计的基于 Express 的文件浏览网站，可以逐级浏览指定目录下的所有文件和文件夹，并针对不同类型文件提供对应的预览、渲染、运行和下载能力。

## 配置项（config.json）
- 端口（默认 8888）
- 根目录（默认 `~/.openclaw/workspace/`）
- Basic Auth 认证（用户名/密码，可选）
- 置顶项列表 `pinnedFolders`
- 跳过的文件/文件夹名称 `skipNames`
- 站点展示配置 `site`
  - `site.title`：站点标题
  - `site.homeEmoji`：面包屑首页图标
  - `site.homeText`：面包屑首页文案
  - `site.rootTitlePrefix`：根目录页面标题前缀

## 认证
- 支持 HTTP Basic Auth（可选）
- 如果配置了用户名和密码，访问任何路径都需要认证
- 留空则无需认证

## 排序规则
1. **置顶项**：按配置的 `pinnedFolders` 顺序显示；支持文件和文件夹
2. **非隐藏文件夹**（按修改时间倒序）
3. **隐藏文件夹**（以 `.` 开头，按修改时间倒序）
4. **非隐藏文件**（按修改时间倒序）
5. **隐藏文件**（以 `.` 开头，按修改时间倒序）

说明：
- 置顶支持任意层级生效，不仅限于首页
- 置顶匹配基于当前浏览项的完整相对路径，例如：
  - `workspace`
  - `workspace/research`
  - `workspace/README.md`
- 未命中的项目按文件夹/文件、隐藏/非隐藏、修改时间规则排序

## 列表项展示信息

### 路径（小字灰色）
- 当前目录下的相对路径显示
- 文件夹：显示文件夹名并追加 `/`
- 文件：显示文件名（含扩展名）

### 标题（大字）
- **Markdown 文件**：提取第一个 H1 标题；没有 H1 则用文件名（去掉扩展名）
- **文件夹**：尝试读取内部 `README.md` 的 H1 标题；没有 README 则用文件夹名
- **其他文件**：直接用文件名
- **长标题**：必须支持自动换行，不能撑破卡片布局

### 介绍（小字灰色）
- Markdown 文件：提取正文前几行作为简介
- 文件夹：使用 `README.md` 的简介
- 其他文件：可为空

## 点击行为
- **文件夹**：进入文件夹，显示内部文件列表
- **Markdown 文件**：渲染显示，支持“原文 / 渲染”切换
- **文本/代码文件**：进入文本预览页，支持语法高亮、文件名复制、下载
- **图片文件**：直接在浏览器中打开
- **音视频 / PDF 文件**：直接在浏览器中打开
- **其他文件**：进入文件详情页，展示元信息并提供下载

## 文本文件预览
- 支持常见代码、配置、日志、纯文本文件的高亮展示
- 顶部显示语言类型和文件大小
- 提供“复制文件名”按钮
- 提供下载按钮
- 大文件可截断显示，避免页面过重

## Markdown 渲染
- 支持 GFM 基础能力
- 支持代码高亮
- 代码块：不换行（`white-space: pre`），左右滚动（`overflow-x: auto`），等宽字体
- 表格：支持横向滚动
- 表格右上角提供“全屏”按钮
- 表格全屏后应从左侧开始完整显示，不得裁掉左边列
- 全屏模式支持横向滚动查看超宽表格
- 支持点击遮罩或按 `Esc` 退出全屏
- 原文/渲染切换按钮：在文章内容区右上角

## 运行按钮（▶ 三角形）
- 如果**文件**扩展名是 `.html`：在该项目右侧垂直居中显示圆形三角形按钮
- 如果**文件夹**包含 `index.html`：同样显示按钮
- 点击按钮：新标签页打开，通过 `/__run` 前缀静态加载
- 按钮样式：圆形、半透明紫色背景、白色 ▶ 图标
- 点击按钮不应触发父层列表项跳转

## 静态文件服务
- `/__run` 前缀挂载 `express.static`，指向配置的根目录
- 主要用于运行 HTML 页面（如小游戏、演示页）

## 下载功能
- 文件详情页提供下载按钮
- 文本文件预览页提供下载按钮
- `/__download/路径` 端点直接下载文件

## 文件详情页
- 展示文件类型、文件大小、修改时间、完整相对路径
- 提供下载按钮
- 提供“复制路径”按钮

## 面包屑导航
- 固定在顶部，一行显示
- 太宽时左右滑动（`overflow-x: auto`, `white-space: nowrap`）
- 从首页开始，逐级显示路径
- 首页图标和文案可由 `site.homeEmoji` / `site.homeText` 配置

## 媒体与资源文件
- 图片文件直接由浏览器展示
- 音频、视频、PDF 直接由浏览器展示
- 其他无法直接预览的文件进入文件详情页

## 安全
- 路径遍历防护：禁止访问根目录外的文件
- 认证保护：HTTP Basic Auth（可选）
- 只读操作：不提供文件上传、修改、删除等写操作

## 技术栈
- Express + EJS + marked + highlight.js
- Node.js 运行
- 绑定地址：`0.0.0.0`
- 端口：配置（默认 8888）

## 运行方式
- 支持直接通过 `node src/server.js` 或 `npm start` 启动
- 可通过 PM2 托管
- PM2 进程名应以当前实际部署配置为准（例如 `workspace-browser`）
