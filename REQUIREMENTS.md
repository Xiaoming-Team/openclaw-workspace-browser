# OpenClaw Workspace 文件浏览器 - 需求文档

## 概述
一个为 OpenClaw 设计的基于 Express 的文件浏览网站，可以逐级浏览指定目录下的所有文件和文件夹。

**配置项（config.json）：**
- 端口（默认 8888）
- 根目录（默认 `~/.openclaw/workspace/`）
- Basic Auth 认证（用户名/密码，可选）
- 固定文件夹列表（首页置顶显示）
- 跳过的文件/文件夹名称

## 认证
- 支持 HTTP Basic Auth（可选）
- 如果配置了用户名和密码，访问任何路径都需要认证
- 留空则无需认证

## 排序规则
1. **置顶文件夹**（仅顶层生效）：从配置的 `pinnedFolders` 中按顺序显示，如果文件夹存在
2. **非隐藏文件夹**（按修改时间倒序）
3. **隐藏文件夹**（以 `.` 开头，按修改时间倒序）
4. **非隐藏文件**（按修改时间倒序）
5. **隐藏文件**（以 `.` 开头，按修改时间倒序）

子目录内不置顶，只按上述 2-5 的分类+时间排序。

## 每个项目的三层信息

### 路径（小字灰色）
- 当前目录下的相对路径
- 文件夹：显示文件夹名
- 文件：显示文件名 + 扩展名

### 标题（大字）
- **Markdown 文件**：提取第一个 H1 标题；没有 H1 则用文件名（去掉扩展名）
- **其他文件**：直接用文件名
- **文件夹**：尝试读取内部 README.md 的 H1 标题；没有 README 则用文件夹名

### 介绍（小字灰色）
- Markdown 文件：提取正文前几行作为简介
- 文件夹：使用 README.md 的简介
- 其他文件：可为空

## 点击行为
- **文件夹**：进入文件夹，显示内部文件列表
- **Markdown 文件**：渲染显示（支持原文/渲染切换）
- **其他文件**：重定向到 `/__run` 路径，由 express.static 处理（根据文件类型自动决定浏览器显示或下载）

## 运行按钮（▶ 三角形）
- 如果**文件**扩展名是 `.html`：在该项目右侧垂直居中显示圆形三角形按钮
- 如果**文件夹**包含 `index.html`：同样显示按钮
- 点击按钮：新标签页打开，通过 `/__run` 前缀静态加载
- 按钮样式：32x32 圆形，半透明紫色背景，白色 ▶ 图标
- 注意：外层容器用 `<div>` + `onclick` 跳转，避免 `<a>` 嵌套 `<a>` 导致按钮位置错乱

## 静态文件服务
- `/__run` 前缀挂载 express.static，指向配置的根目录
- 用于运行按钮打开 HTML 页面、以及非MD文件的浏览/下载

## 下载功能
- 文件详情页提供"下载"按钮
- `/__download/路径` 端点直接下载文件

## 面包屑导航
- 固定在顶部，一行显示
- 太宽时左右滑动（overflow-x: auto, white-space: nowrap）
- 从 🏠 首页 开始，逐级显示路径

## Markdown 渲染
- 代码块：不换行（white-space: pre），左右滚动（overflow-x: auto），等宽字体
- 表格：支持全屏按钮（横屏查看）
- 原文/渲染切换按钮：在文章内容区右上角，浅灰背景+深色文字

## 安全
- 路径遍历防护：禁止访问根目录外的文件
- 认证保护：HTTP Basic Auth（可选）
- 只读操作：不提供文件上传、修改、删除等写操作

## 技术栈
- Express + EJS + marked + highlight.js
- PM2 管理，服务名 openclaw-workspace-browser
- 绑定地址：0.0.0.0
- 端口：配置（默认 8888）
